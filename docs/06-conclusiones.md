# SECCIÓN 6. CONCLUSIONES Y REFLEXIÓN CRÍTICA

## Descripción

La etapa final del proyecto consolida los resultados obtenidos durante el desarrollo del Sistema de Gestión de Reservas y Operaciones para el Restaurante D'Barrio Broaster, evaluando el cumplimiento de los objetivos planteados, las decisiones técnicas adoptadas y las oportunidades de mejora identificadas durante el proceso de desarrollo.

Asimismo, esta sección presenta una valoración crítica del proyecto, documentando las principales conclusiones, las limitaciones del producto implementado y la reflexión individual de cada integrante del equipo, permitiendo evidenciar los aprendizajes obtenidos y la experiencia adquirida durante el desarrollo del sistema. 

## Conclusiones del proyecto

Los resultados obtenidos demuestran que la aplicación del modelo Scrum adaptado permitió desarrollar el sistema de manera incremental, organizando el trabajo mediante Sprints y validando progresivamente cada módulo funcional antes de su incorporación al producto final.

El sistema integra en una única plataforma los procesos de reserva, recepción, administración de mesas, lista de espera, gestión de pedidos, control de caja, administración de usuarios y generación de reportes, mejorando la organización de la información y fortaleciendo la trazabilidad entre las operaciones realizadas por los diferentes usuarios del restaurante.

Desde el punto de vista técnico, la utilización de una arquitectura basada en el patrón Modelo–Vista–Controlador (MVC), junto con mecanismos de autenticación, autorización por roles, protección CSRF y acceso seguro mediante PDO, proporciona una estructura mantenible y alineada con las buenas prácticas para el desarrollo de aplicaciones web. 

## Limitaciones identificadas

El alcance definido para el proyecto corresponde a una implementación académica, por lo que determinadas funcionalidades fueron consideradas como oportunidades de evolución para futuras versiones del sistema.

Entre las principales limitaciones identificadas se encuentran:

- La validación de adelantos continúa realizándose mediante revisión manual de vouchers.
- No se implementó integración con pasarelas de pago electrónico.
- El sistema no incorpora servicios automáticos de notificación mediante correo electrónico o mensajería.
- No se desarrolló una aplicación móvil nativa.
- No se implementaron pruebas automatizadas ni procesos de integración continua (CI).
- El despliegue se encuentra orientado a un entorno local utilizando Apache, PHP y MySQL/MariaDB.

Estas limitaciones no afectan el cumplimiento de los objetivos definidos para el proyecto, pero representan oportunidades de mejora para futuras etapas de desarrollo. 

## Reflexión crítica

El desarrollo del proyecto permitió aplicar de manera integrada conocimientos relacionados con análisis de requisitos, planificación ágil, diseño de software, implementación, aseguramiento de la calidad y documentación técnica.

La experiencia obtenida durante el desarrollo evidencia la importancia de mantener una comunicación constante con los usuarios representativos, validar continuamente los incrementos funcionales y documentar las decisiones adoptadas durante todo el ciclo de vida del software.

Las reflexiones individuales complementan esta evaluación describiendo la participación de cada integrante, las decisiones técnicas asumidas, las dificultades encontradas y las mejoras que podrían incorporarse en futuros proyectos de desarrollo de software.
