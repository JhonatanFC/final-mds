# SECCIÓN 3. GESTIÓN DEL PROYECTO

## Descripción

La gestión del proyecto se planificó siguiendo el modelo Scrum adaptado, organizando el desarrollo mediante iteraciones cortas que permitieron entregar incrementos funcionales, validar continuamente los resultados y mantener la trazabilidad entre los requisitos del negocio y las funcionalidades implementadas.

El proyecto se desarrolló dentro del calendario académico establecido, considerando actividades de planificación, análisis, diseño, implementación, pruebas y validación, asegurando que cada Sprint generara un incremento verificable antes de iniciar la siguiente iteración. La planificación permitió distribuir el trabajo de forma progresiva, reducir riesgos durante el desarrollo y mantener un control permanente sobre el avance del proyecto. :contentReference[oaicite:1]{index=1}

## Cronograma del proyecto

La ejecución del proyecto se estructuró en cinco Sprints consecutivos, cada uno orientado a desarrollar un conjunto específico de funcionalidades relacionadas con la operación del restaurante.

|---------|--------------------|----------------------|
| Sprint 0 | Preparación del entorno, planificación y organización del Product Backlog. | 10/05/2026 |
| Sprint 1 | Implementación de autenticación, usuarios y configuración inicial del sistema. | 24/05/2026 |
| Sprint 2 | Desarrollo del módulo de reservas, recepción y gestión de mesas. | 07/06/2026 |
| Sprint 3 | Implementación de pedidos, productos y flujo operativo del restaurante. | 21/06/2026 |
| Sprint 4 | Desarrollo del módulo de caja, reportes, validaciones finales y preparación de la entrega. | 05/07/2026 |

Cada Sprint concluye con una Sprint Review orientada a verificar el cumplimiento de los criterios de aceptación definidos para las historias de usuario desarrolladas. Posteriormente, el equipo realiza una Sprint Retrospective con el propósito de identificar oportunidades de mejora para la siguiente iteración. :contentReference[oaicite:2]{index=2}

## Gestión de recursos

El desarrollo del proyecto se realizó utilizando herramientas de software libre y un entorno de implementación local basado en PHP, Apache y MySQL/MariaDB. La organización del trabajo permitió optimizar el tiempo disponible durante el ciclo académico, asignando responsabilidades específicas a cada integrante del equipo y manteniendo una coordinación permanente durante todas las etapas del desarrollo.

La planificación de recursos también contempló el esfuerzo destinado al análisis de requisitos, diseño del sistema, construcción de la aplicación, ejecución de pruebas, elaboración de documentación y validación del producto final. :contentReference[oaicite:3]{index=3}

## Gestión de riesgos

Durante la planificación se identificaron riesgos relacionados con cambios en los requisitos, disponibilidad de los usuarios para validar incrementos, restricciones de tiempo propias del calendario académico y posibles errores durante la integración de los módulos del sistema.

Para reducir su impacto se establecieron acciones preventivas como la planificación incremental, revisiones periódicas del Product Backlog, validaciones funcionales en cada Sprint, seguimiento continuo del avance y aplicación de pruebas antes de aprobar cada incremento desarrollado.

La gestión de riesgos permitió mantener el proyecto dentro del alcance previsto y disminuir la probabilidad de retrabajo durante las fases finales del desarrollo. :contentReference[oaicite:4]{index=4}

## Seguimiento y control

El control del proyecto se realizó mediante indicadores definidos desde el inicio del desarrollo, permitiendo evaluar objetivamente el progreso de cada Sprint y la calidad del producto obtenido.

Entre los principales indicadores utilizados se encuentran:

- Cumplimiento de historias de usuario por Sprint.
- Velocity del equipo de desarrollo.
- Porcentaje de casos de prueba aprobados.
- Número de defectos críticos detectados.
- Cobertura de trazabilidad entre requisitos, diseño, implementación y pruebas.
- Tiempo promedio de respuesta de las funcionalidades principales.

El seguimiento continuo de estos indicadores permitió verificar el cumplimiento de los objetivos establecidos para cada iteración y facilitó la toma de decisiones durante las reuniones de seguimiento del proyecto. Los resultados detallados de las métricas, el cronograma completo, la planificación de Sprints, el presupuesto y los registros de riesgos se presentan en la documentación técnica correspondiente y en los anexos del proyecto. :contentReference[oaicite:5]{index=5}
