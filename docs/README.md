Cada elemento terminado apunta a archivos existentes en el repositorio. Los eventos Scrum históricos que no cuentan con actas o commits no se presentan como hechos. Desde la publicación del repositorio, las futuras iteraciones deberán documentarse mediante issues, ramas, commits y pull requests.
