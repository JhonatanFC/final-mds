# Documentación del Proyecto

## Sistema de Gestión de Reservas y Operaciones para el Restaurante D'Barrio Broaster (RMRS)

Esta carpeta reúne la documentación técnica, metodológica y de aseguramiento de la calidad desarrollada para el proyecto **Restaurant Management & Reservation System (RMRS)**. Su finalidad es organizar de forma estructurada toda la evidencia generada durante el ciclo de vida del software, permitiendo comprender las decisiones tomadas desde el análisis inicial hasta la validación del producto final.

La documentación complementa el informe académico principal y facilita la consulta independiente de cada uno de los artefactos producidos durante el proyecto. Cada documento mantiene la trazabilidad con los requisitos del negocio, el modelo de proceso seleccionado, el diseño del sistema, las actividades de aseguramiento de la calidad y los resultados obtenidos durante el desarrollo.

## Estructura de la documentación

La documentación técnica se encuentra organizada en las siguientes secciones:

1. **Diagnóstico de la organización**  
   Presenta el contexto organizacional, el análisis del problema, las necesidades identificadas y el alcance del sistema.

   → [01-diagnostico.md](01-diagnostico.md)

2. **Selección y justificación del modelo de proceso**  
   Describe la evaluación de alternativas, la selección de Scrum adaptado y la planificación general del proyecto.

   → [02-modelo-proceso.md](02-modelo-proceso.md)

3. **Gestión del proyecto**  
   Documenta la planificación de Sprints, la administración de recursos, la gestión de riesgos y los mecanismos de seguimiento y control.

   → [03-gestion-proyecto.md](03-gestion-proyecto.md)

4. **Documento de Diseño de Software (SDD)**  
   Describe la arquitectura del sistema, el diseño modular, el modelo de datos, la navegación, la seguridad, la persistencia y las decisiones técnicas adoptadas durante el desarrollo.

   → [04-diseno-software.md](04-diseno-software.md)

5. **Plan de Calidad del Software**  
   Presenta el perfil de calidad basado en ISO/IEC 25010, el proceso de Software Quality Assurance (SQA), la estrategia Shift Left y las actividades de verificación y validación realizadas durante el proyecto.

   → [05-calidad.md](05-calidad.md)

6. **Conclusiones y reflexión crítica**  
   Resume los resultados obtenidos, las principales conclusiones, las limitaciones identificadas y las oportunidades de mejora del sistema.

   → [06-conclusiones.md](06-conclusiones.md)

7. **Documento de Diseño de Software (SDD) completo**  
   Contiene el desarrollo técnico integral del diseño del sistema y sirve como documento de referencia para la implementación y mantenimiento del software.

   → [sdd.md](sdd.md)

## Organización de la información

La estructura del repositorio mantiene correspondencia con el contenido del informe académico entregado para el proyecto. Cada documento desarrolla una sección específica del proceso de ingeniería de software, permitiendo consultar de forma independiente el análisis, la planificación, el diseño, la implementación y el aseguramiento de la calidad sin perder la trazabilidad entre los diferentes artefactos.

## Consideraciones

Toda la documentación corresponde a un proyecto académico desarrollado para el diseño e implementación de un sistema de gestión de reservas y operaciones para un restaurante ficticio. Los cronogramas, presupuestos, métricas e indicadores presentados fueron elaborados con fines de planificación y validación académica, manteniendo coherencia con el alcance definido para el proyecto y con el funcionamiento del sistema implementado dentro de este repositorio.
