# Anexo H. Guion de Sprint Review

## Introducción

El presente guion documenta los resultados obtenidos durante las reuniones de **Sprint Review** realizadas a lo largo del desarrollo del Sistema de Gestión de Reservas y Operaciones para el Restaurante D'Barrio Broaster.

Cada revisión tuvo como propósito presentar el incremento desarrollado durante el sprint correspondiente, verificar el cumplimiento de las historias de usuario comprometidas, recopilar observaciones del Product Owner y validar que las funcionalidades implementadas cumplían con los criterios de aceptación definidos durante la planificación.

La información presentada resume los principales resultados obtenidos en cada Sprint Review y las acciones de mejora identificadas para los siguientes incrementos del proyecto.

## Detalle del Sprint 0

| Campo | Descripción |
|--------|-------------|
| **Objetivo** | Implementar la base técnica del sistema y establecer la arquitectura inicial del proyecto. |
| **Historias de Usuario** | HU-01, HU-02 |
| **Evidencia del incremento** | Arquitectura MVC implementada, configuración del proyecto, autenticación, gestión de usuarios y control de acceso. |
| **Sprint Review** | **Fecha:** 24/05/2026. **Resultado:** Incremento aprobado tras validar el funcionamiento de la autenticación, la conexión con la base de datos y la estructura general del sistema. |
| **Sprint Retrospective** | **Se mantiene:** Organización modular del proyecto. **Se mejora:** Documentación técnica inicial. **Acción:** Estandarizar la estructura del código y fortalecer la documentación desde las primeras iteraciones. |

## Detalle del Sprint 1

| Campo | Descripción |
|--------|-------------|
| **Objetivo** | Desarrollar el proceso inicial de reservas y recepción del cliente. |
| **Historias de Usuario** | HU-03, HU-04, HU-05, HU-06 |
| **Evidencia del incremento** | Registro de reservas, validación de vouchers, asignación de mesas y lista de espera. |
| **Sprint Review** | **Fecha:** 07/06/2026. **Resultado:** Incremento aprobado tras comprobar el correcto funcionamiento del flujo de reservas y la administración de mesas. |
| **Sprint Retrospective** | **Se mantiene:** Validación continua con los usuarios del sistema. **Se mejora:** Experiencia de usuario durante el proceso de recepción. **Acción:** Incorporar mensajes de validación más claros y optimizar la navegación del módulo de reservas. |

## Detalle del Sprint 2

## Conclusión

Las reuniones de Sprint Review permitieron validar cada incremento funcional desarrollado durante el proyecto, asegurando que las historias de usuario implementadas cumplieran con los objetivos establecidos al inicio de cada sprint. Asimismo, las observaciones registradas durante las revisiones sirvieron como insumo para la mejora continua del producto y del proceso de desarrollo, manteniendo la alineación entre los requerimientos del proyecto y el sistema implementado.
