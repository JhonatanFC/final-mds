# Anexo B. Detalle de sprints

La siguiente tabla resume la planificación y ejecución de los cinco sprints desarrollados durante el proyecto. En ella se presenta el objetivo de cada incremento, las historias de usuario implementadas, el incremento entregado, los resultados obtenidos durante el Sprint Review y las acciones de mejora identificadas en la Sprint Retrospective, manteniendo la misma estructura presentada en el informe final.

| Sprint | Objetivo | Historias de Usuario | Incremento entregado | Resultado Sprint Review | Acción de mejora (Retrospectiva) |
|:------:|-----------|----------------------|----------------------|--------------------------|----------------------------------|
| **Sprint 0** | Implementar la base técnica del sistema. | HU-01, HU-02 | Arquitectura MVC, configuración del proyecto, base de datos, autenticación, gestión de usuarios y control de acceso. | **Aprobado.** Se verificó el correcto funcionamiento de la arquitectura base, la autenticación y la conexión con la base de datos. | Estandarizar la estructura del código y fortalecer la documentación técnica desde las primeras iteraciones. |
| **Sprint 1** | Desarrollar el proceso inicial de atención al cliente. | HU-03, HU-04, HU-05, HU-06 | Módulo de reservas, recepción, asignación de mesas y lista de espera. | **Aprobado.** Las funcionalidades permitieron registrar y administrar reservas correctamente. | Mejorar la experiencia de usuario mediante mensajes de validación más claros y una navegación más intuitiva. |
| **Sprint 2** | Implementar la gestión operativa del restaurante. | HU-07, HU-08 | Administración de productos, categorías y módulo de pedidos. | **Aprobado.** Se comprobó el registro correcto de pedidos y la administración del catálogo de productos. | Optimizar las consultas a la base de datos y mejorar el rendimiento del módulo de pedidos. |
| **Sprint 3** | Implementar el proceso de caja y cobros. | HU-09 | Registro de caja, aplicación del adelanto y cálculo del saldo final del pedido. | **Aprobado.** El proceso de cobro se ejecutó correctamente aplicando el adelanto registrado durante la reserva. | Reforzar las validaciones entre reservas, pedidos y cobros para garantizar la consistencia de la información. |
| **Sprint 4** | Implementar el dashboard y los reportes gerenciales. | HU-10 | Dashboard, reportes e integración de la documentación final del proyecto. | **Aprobado.** Se verificó la correcta generación de indicadores y reportes para apoyar la toma de decisiones. | Continuar fortaleciendo la mantenibilidad del sistema y documentar futuras mejoras para nuevas versiones. |

**Fuente:** Elaboración propia.
