# Anexo K. Lista de verificación previa a la sustentación

## Introducción

La presente lista de verificación tiene como finalidad confirmar que toda la documentación, los artefactos del proyecto y el producto desarrollado mantienen coherencia antes de la sustentación final. Su revisión permite identificar oportunamente posibles inconsistencias entre el informe, el repositorio, el prototipo, las evidencias y la demostración del sistema, reduciendo observaciones durante la evaluación.

## Lista de verificación

| N.° | Verificación | Estado |
|:---:|--------------|:------:|
| 1 | El documento menciona únicamente módulos presentes en el producto final. | ☐ |
| 2 | El Excel de sprints coincide con el Product Backlog y las fechas expuestas. | ☐ |
| 3 | El enlace de Figma funciona y contiene las pantallas correspondientes. | ☐ |
| 4 | Los diagramas presentan título, numeración y la fuente **"Elaboración propia"**. | ☐ |
| 5 | La demostración del sistema incluye el flujo de reserva, recepción, pedido, caja y reportes. | ☐ |
| 6 | Cada integrante conoce su contribución y puede explicar las decisiones técnicas adoptadas durante el desarrollo. | ☐ |
| 7 | Las referencias bibliográficas cumplen el formato APA 7 y la declaración de autoría se encuentra firmada. | ☐ |

## Observación

Esta lista de verificación debe revisarse el día anterior a la sustentación final. Su aplicación permite reducir inconsistencias entre la exposición, el informe académico, el repositorio del proyecto y el producto ejecutable, asegurando que todos los entregables mantengan coherencia y correspondencia con la documentación presentada.
