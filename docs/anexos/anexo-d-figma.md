# Anexo D. Enlace de diseño Figma

## Prototipo del sistema

El prototipo de alta fidelidad del **Sistema de Gestión de Reservas y Operaciones para el Restaurante D'Barrio Broaster** fue desarrollado en Figma como parte del proceso de diseño de interfaces de usuario. Su elaboración permitió definir la distribución visual de las pantallas, la navegación entre módulos y la interacción esperada para cada uno de los procesos implementados en el sistema.

El diseño fue realizado siguiendo los criterios de usabilidad establecidos en el Documento de Diseño de Software, procurando mantener una interfaz intuitiva, consistente y alineada con los requerimientos funcionales del proyecto.

## Contenido del prototipo

El prototipo deberá incluir, como mínimo, las siguientes pantallas desarrolladas durante el proyecto:

- Inicio de sesión.
- Registro de reservas.
- Validación de vouchers.
- Gestión de mesas.
- Lista de espera.
- Registro de pedidos.
- Gestión de productos.
- Módulo de caja.
- Administración de usuarios.
- Dashboard y reportes gerenciales.

Todas las pantallas deberán mantener coherencia con el diseño de navegación, la arquitectura del sistema y los módulos implementados en el producto final.
