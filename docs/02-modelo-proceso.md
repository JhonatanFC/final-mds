# Sección 2. Selección y justificación del modelo de proceso

Se seleccionó **Scrum adaptado** para entregar valor por incrementos verificables y revisar cada módulo con usuarios representativos. La adaptación incorpora Product Backlog, criterios de aceptación, Sprint Review, retrospectiva, matriz de pruebas, riesgos y trazabilidad.

Se evaluaron Cascada, Incremental y Scrum. Scrum obtuvo la mayor adecuación porque las reglas principales son conocidas, pero los flujos de reservas, vouchers, mesas, pedidos y caja necesitan retroalimentación frecuente.

La comparación contextual y los roles se documentan en [modelo de proceso](entrega/03-modelo-de-proceso.md).
